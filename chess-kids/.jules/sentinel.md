## 2025-04-24 - Add DoS Protection to AI Tutor API
**Vulnerability:** The `/api/tutor` endpoint and utility functions lacked validation limits on the length and count of incoming messages, as well as the length of the system prompt. This omission could lead to resource exhaustion and potential Denial of Service (DoS) attacks by sending excessively large payloads to the AI model provider.
**Learning:** External API dependencies, particularly AI model generation endpoints, must always bounds-check incoming user data. Allowing unbounded arrays or strings invites excessive computational costs and potential outages. Due to environment constraints where `server.js` (an ES Module) cannot natively import TypeScript utilities without a build step, duplication of validation logic is sometimes necessary to guarantee local environment parity.
**Prevention:** Always set explicit maximum bounds on array lengths and string sizes for data submitted from clients, particularly before sending such data to rate-limited or compute-heavy third-party services. Validate constraints like allowed enum values (e.g., 'user', 'assistant') early in the pipeline.
