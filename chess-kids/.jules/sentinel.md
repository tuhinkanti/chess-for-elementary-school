## 2025-04-27 - [MEDIUM] Fix missing input validation (DoS risk)
**Vulnerability:** The AI tutor API endpoints (`api/tutor.ts` and `server.js`) lacked length constraints on arrays and string inputs. Specifically, the endpoints did not validate the maximum number of conversation messages, or the character limits of individual user messages or system prompts.
**Learning:** External AI APIs consume tokens proportionally to the size of the request. Permitting infinitely large inputs or excessively long arrays opens the system up to Resource Exhaustion and DoS attacks, alongside spiraling token consumption costs from upstream providers. Additionally, Node.js cannot natively resolve TypeScript files imported into plain JavaScript ES Modules like `server.js` without a build step (resulting in `ERR_MODULE_NOT_FOUND`). Shared logic such as `validateTutorRequest` had to be duplicated within `server.js` rather than imported from `src/utils/aiUtils.ts`.
**Prevention:** Always enforce strict lengths, bounds, and array sizes at the boundary/controller level before sending data to external APIs. Length checks must be applied consistently to array bounds as well as text bodies to effectively mitigate DoS attacks.
