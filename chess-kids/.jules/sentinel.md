## 2024-04-10 - Missing input limits on AI API endpoints
**Vulnerability:** The `/api/tutor` endpoint and local `server.js` lacked limits on the size of the messages array and the length of individual message contents, exposing the application to DoS attacks via resource exhaustion and potential excessive API token usage.
**Learning:** LLM and AI APIs must always have strict bounds on arrays and string attributes (e.g. max 50 messages, max 1000 chars per message), as well as validating strictly expected enum strings (like roles), to prevent both DoS and prompt manipulation. ESM constraint meant shared logic from `utils/aiUtils.ts` couldn't be cleanly imported into Node.js `server.js` without a build step, requiring validation replication.
**Prevention:** Always enforce explicit input length restrictions, bounds, and role validation on any user input interacting with external AI APIs. Ensure duplicated validation paths (like serverless functions vs. dev servers) are kept in sync when sharing code is restricted.
