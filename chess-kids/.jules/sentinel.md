## 2023-10-27 - Prompt Injection via Client-Side System Prompt Override
**Vulnerability:** The AI tutor endpoints (`api/tutor.ts` and `server.js`) accepted an arbitrary `systemPrompt` text directly from the client request payload and used it to guide the LLM's behavior without sanitization or validation. This allowed an attacker to completely override the intended constraints (such as forcing JSON responses or defining the persona) by passing a malicious `systemPrompt` value.
**Learning:** Security controls related to LLM generation (such as instructions enforcing response formatting or mitigating harmful responses) must be constructed and enforced strictly on the server-side. Relying on the client to securely communicate the required system prompt is an instance of CWE-602 (Client-Side Enforcement of Server-Side Security).
**Prevention:** Rather than trusting the client to format and supply the `systemPrompt`, the client should send structured context (such as game state or objective info). The backend must then parse this context and securely synthesize the final `systemPrompt` internally before dispatching the request to the AI provider.
