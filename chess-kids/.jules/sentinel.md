## 2025-05-03 - AI Prompt Validation DoS Risk
**Vulnerability:** Missing bounds validation on user-supplied parameters (message count, system prompt length, message length) and missing role verification when interacting with the AI API. Attackers could submit massive payloads or invalid roles causing memory exhaustion, unexpected AI parsing behavior, and denial of service (DoS).
**Learning:** External AI API integration points are primary attack vectors for resource exhaustion. Simply verifying `typeof body === 'object'` and extracting an unbounded array of messages is insufficient. `server.js` and `api/tutor.ts` both lacked robust bounds checking.
**Prevention:** Always enforce explicit, strict bounds (e.g. maximum number of items, maximum string lengths) on ALL external inputs before they reach memory-intensive operations. Also strictly validate enums (like chat 'roles'). Where possible, use schema validation libraries like Zod instead of manual iteration to guarantee type and bounds safety.
