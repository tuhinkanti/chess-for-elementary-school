## 2024-04-14 - Fix Missing Input Length Limits (DoS Risk)
**Vulnerability:** The AI Tutor endpoints (`api/tutor.ts` and `server.js`) accepted user-provided `messages` arrays and `systemPrompt` strings without enforcing any length limitations. This created a Denial of Service (DoS) and resource exhaustion risk, as an attacker could send arbitrarily large arrays or extremely long text, consuming excessive AI tokens, increasing cloud provider costs, and potentially overwhelming backend memory.
**Learning:** Even internal or seemingly safe AI integrations require strict input bounds. Generative AI tools are particularly vulnerable to resource exhaustion attacks if the input length is not constrained. Schema validation (e.g. max length, max items) is necessary to protect against both malicious attacks and accidental misuse.
**Prevention:** Always implement explicit input length and size limitations (e.g., maximum string length, maximum array size) on all endpoints, particularly those interacting with external APIs or resource-intensive operations like LLMs. Use robust validation logic early in the request lifecycle.
