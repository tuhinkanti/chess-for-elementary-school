## 2025-05-14 - Fix missing input limits (DoS risk)
**Vulnerability:** The AI tutor request validation (`validateTutorRequest`) lacked bounds checking on the size of the `messages` array and the length of the string `content` inside each message, allowing an attacker to submit arbitrarily large payloads. This can lead to Resource Exhaustion and Denial of Service (DoS), especially when hitting external AI APIs which could incur excessive costs or quota exhaustion. It also failed to validate the `role` property.
**Learning:** Even internal or semi-internal API utilities must implement strong guardrails on array sizes, string lengths, and enumerate permitted values. Relying solely on type assertions (e.g. `b.messages as any[]`) without runtime structure validation leaves the application exposed to malformed data.
**Prevention:** Always implement explicit input length limits (e.g. max 50 messages, max 1000 characters per message) and strict structural validation for all inputs that are passed to external services or expensive operations.
