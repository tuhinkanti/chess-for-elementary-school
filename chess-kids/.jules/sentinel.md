## 2025-04-15 - [DoS Risk] Missing Length Limits on AI API Inputs
**Vulnerability:** The AI tutor API endpoints (`api/tutor.ts` and `server.js`) accepted user-provided chat messages and system prompts without any length or quantity limits. This allowed an attacker to submit massive payloads to the `generateText` API, causing excessive token consumption, high API costs, and potential Denial of Service (DoS).
**Learning:** External AI APIs are billed by token and consume significant backend resources. Failing to set explicit boundaries on arrays (number of messages) and string attributes (message lengths) submitted to these APIs creates a critical resource exhaustion vector. Validation logic must also be duplicated in plain JS environments (`server.js`) that cannot import TypeScript modules directly.
**Prevention:** Always implement strict input validation on all AI endpoints. Set maximum limits for array lengths (e.g., max 50 messages), string lengths (e.g., max 1000 chars per message), and validate specific roles to prevent malicious data ingestion and resource abuse.
