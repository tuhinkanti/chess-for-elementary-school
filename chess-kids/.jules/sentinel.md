## 2024-04-28 - Missing length constraints in array / string properties causes DoS vulnerability
**Vulnerability:** The `validateTutorRequest` in `aiUtils.ts` and `server.js` verified the format of the incoming request body, but did not impose any length limits. It accepted any number of messages with arbitrarily long text strings, resulting in possible resource exhaustion, excess token usage, or denial-of-service (DoS) via prompt injection or oversized payloads.
**Learning:** Always implement strict maximum limits on array sizes and string fields from external API requests, particularly when those payloads are subsequently relayed to expensive or constrained third-party services like LLMs.
**Prevention:** In `validateTutorRequest`, enforced a strict limit of 50 `messages`, a limit of 1000 characters for message `content`, and 2000 characters for `systemPrompt`. Included explicit type and length tests to automatically verify validation rules to prevent DoS going forward.
