## 2024-05-18 - [Resource Exhaustion via Unbounded Input in AI Tutor API]
**Vulnerability:** The AI Tutor API (`/api/tutor` and `server.js`) accepted an arbitrary number of chat messages with unbounded length, allowing a potential denial-of-service (DoS) or resource exhaustion attack on the backend infrastructure and LLM providers.
**Learning:** Type checking (e.g., verifying `messages` is an array) is insufficient for security. Deep, schema-level validation is required to ensure constraints like maximum array size, allowed string enumerations (e.g., `role`), and strict string length limits are enforced. The application level is often the final defense against malicious inputs targeting third-party APIs.
**Prevention:** Implement strict validation of all fields in request payloads, especially for data sent to expensive or rate-limited third-party services. In this case, ensuring maximum lengths for `messages` array (e.g., 50) and `content` (e.g., 1000 chars), validating role strings, and applying Express JSON payload limits (`express.json({ limit: '1mb' })`).
