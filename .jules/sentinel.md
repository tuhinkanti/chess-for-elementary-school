## 2024-05-14 - Add prompt message role and length validation to AI API
**Vulnerability:** The AI tutor request validation (`validateTutorRequest`) lacked checks on message roles and message content length. This exposed the application to possible prompt injection attacks using unauthorized roles (e.g. `system` spoofing) and potential Denial of Service (DoS) via excessive tokens since long messages would consume significant compute and token quotas.
**Learning:** Even if `tutor.ts` serverless function filtered out `system` roles, standard validation for all AI prompt interactions should strictly define allowed inputs at the earliest layer. Missing input validation specifically tailored for LLM bounds (token limits, valid conversation roles) is a common oversight that impacts both security and application reliability.
**Prevention:** Implement strict schema validation on all API endpoints feeding into LLMs. Limit message lengths to a reasonable character maximum (e.g., 1000 characters) and explicitly allowlist valid message roles (`user`, `assistant`, `system`).
