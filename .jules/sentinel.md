## 2026-04-25 - [Strict LLM Input Boundaries]
**Vulnerability:** The AI tutor endpoints (in `api/tutor.ts` and `server.js`) lacked boundary constraints (e.g., maximum array length, individual string sizes) on user input. This created a Denial of Service (DoS) and resource exhaustion vulnerability where an attacker could send excessively large prompt arrays or strings to the backend LLM service.
**Learning:** Due to Node.js module resolution limits where TS utilities couldn't be easily imported by standard `server.js` without build tooling, input validation had to be meticulously duplicated across both the Vercel serverless functions and the local express server to prevent vulnerabilities from sneaking into distinct backend execution environments. Unbounded arrays or message contents must always be strictly limited before being passed to external AI SDKs to cap maximum request payloads.
**Prevention:** Always enforce strict length and size limits on array structures (`messages.length <= 50`) and individual string properties (`content.length <= 1000`, `systemPrompt.length <= 2000`) when passing user data directly to LLM APIs. Furthermore, ensure these constraints are symmetrically enforced across all application deployment targets.
