## 2024-03-15 - [CRITICAL] Fix Prompt Injection and Missing Input Limits in AI Tutor API
**Vulnerability:** The AI Tutor API (`/api/tutor`) blindly accepted `systemPrompt` from the client request payload without verifying or sanitizing it. This allowed for prompt injection where a malicious user could pass a payload to ignore instructions and act maliciously. Additionally, `validateTutorRequest` did not enforce length limits or validate role types properly, exposing the application to token-based Denial of Service (DoS) attacks and logic bypasses.
**Learning:** Security boundaries must be respected between the client and server. The client should only pass state or context (e.g., `GameContext`), and the server must exclusively handle the construction of security-sensitive components like the system prompt. Relying on the client to send a correct and safe `systemPrompt` breaks the principle of zero-trust.
**Prevention:** Do not accept `systemPrompt` from client requests. Construct system prompts server-side using trusted data (like context variables) and shared utility functions. Always validate message structures, enforce maximum string length limits (e.g., 1000 characters) on user input, and restrict roles strictly to `['user', 'assistant', 'system']` during request validation.
